Detailed Description

Mutex operation tracker.

Records precise timing of mutex state transitions with nanosecond resolution for contention analysis.

Stores data as [request_time, acquire_time, release_time] tuples indexed by mutex name.

Parameters

Exceptions
std::runtime_errorif attempting to record a state transition without existing log
std::invalid_argumentif provided an invalid ThreadMutexState value
Note
  • Each mutex log contains three timestamps corresponding to:
    1. NEED (request time)
    2. HAVE (acquisition time)
    3. FREE (release time)
  • State transitions must follow NEED->HAVE->FREE sequence
